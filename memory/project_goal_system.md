## 2026 Goal System

**Annual Theme:** "Year of Being Grounded" — foundation, presence, intention
**Decision Filter:** "Does this ground me, or does this scatter me?"

### 8 Goals:
1. Career: From Grind to Growth (stay-or-go by end Q2)
2. Marriage & Home: Grounded Partnership (joint financial vision, couple rituals, family planning)
3. Family Roots: Show Up Before It's Too Late (parents 2x/month, sister support, legacy interviews every 6 weeks)
4. Friendships: Invest Where It's Mutual (core circle + professional network rebuild)
5. Physical Health: Build a Body That Lasts (lifting, mobility, cholesterol, sleep, alcohol 1x/month max)
6. Inner Life: Cultivate Stillness and Joy (meditation, reading 1 book/month, DJ, nature 2x/week)
7. Finances: Build Wealth with Intention (advisor, joint plan with Bonnie, creative investments, $15-20M target by ~50)
8. Personal Brand: Ship and Share (website by end of April, 1 post/week, 2 AI projects by end Q2)

### Review Cadence:
- Daily: /morning-plan (surfaces goals + lead indicators) + /reflect (goal-aligned rescheduling)
- Weekly: /weekly-review (execution scoring — % of This Week targets achieved)
- Monthly: /monthly-review (RAG status, trajectory check, anti-goal audit)
- Quarterly: /quarterly-planning (score 0.0-1.0, retrospective, set next quarter with WOOP)

### Key Design Decisions:
- Single quarterly file per quarter (not individual goal files) — all skills read one file
- WOOP (Obstacle + If-Then) inline per goal
- Lead indicators = weekly trackable inputs per goal
- Execution scoring = 12 Week Year method (target 85%+)
- Anti-goals tracked in annual file, checked in monthly review

**Why:** Research-backed composite of OKRs, 12 Week Year, WOOP, EOS Rocks, and identity-based goal setting. Evidence: specific+difficult goals (d=0.42-0.82), implementation intentions (d=0.65), progress monitoring (d=0.40).

**How to apply:** When running any POS skill, reference the quarterly goals file for context. When proposing tasks or priorities, check goal alignment. When rescheduling, use WOOP obstacles as context.
