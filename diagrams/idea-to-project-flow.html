Idea to Project Flow

Personal OS pipeline — from thought to scaffolded project with context, agents, and Obsidian links

Stage 1 Capture /idea

Talk about an idea freely. Claude synthesizes it into a structured note with topics, tags, and vault connections. 30 seconds from thought to linked note.

Ideas/{slug}.md status: seed topics + tags vault connections
Obsidian I/O ← Reads: People/, Resources/, Projects/, Meetings/ → Writes: Ideas/{slug}.md
Stage 2 Incubate /weekly-review

Ideas sit as seeds. Weekly review surfaces ideas older than 14 days and asks: promote, archive, or leave? No pressure — ideas are cheap.

seed idea surfaced promote / archive / keep
Obsidian I/O ← Reads: Ideas/ (all seed status) ↔ May update: status → archived
Stage 3 Promote /promote

Select an idea to turn into a project. Confirm slug, area, stakeholders, goal link, and whether it needs a code directory.

project slug confirmed area: work | personal goal linked scaffold decision
Obsidian I/O ← Reads: Ideas/{slug}.md ← Reads: Goals/2026-Q2.md
Stage 4 Kickoff /promote (step 4)

Conversational deep-dive with Claude. Develops scope, architecture, constraints, users, and key decisions. This conversation becomes the CLAUDE.md — the project's brain.

CLAUDE.md (rich, generated) scope defined architecture direction key decisions captured
Obsidian I/O ← Reads: idea Shape + Open Questions ← Reads: People/ (stakeholders)
Stage 5 Hire Agents /promote (step 5)

Scan all projects for available agents. Present them as a hiring menu. Select existing agents to bring on, or describe new ones to create.

agents/{name}.md (per hire) agents/registry.md ~/.claude/agents/ (if global) registry updated
Registry I/O ← Reads: ~/.claude/project-registry.md ← Reads: ~/.claude/agents/ ← Reads: ~/projects/*/agents/
Stage 6 Scaffold /promote (steps 6-11)

Create everything: Obsidian project note (pre-filled from idea), code directory with CLAUDE.md + agents + git init, update idea status, link goal, update registry.

Projects/{slug}.md ~/projects/{slug}/ CLAUDE.md .claude/skills/ .claude/commands/ agents/registry.md .gitignore + README project-registry.md idea status → project goal linked
Obsidian I/O → Writes: Projects/{slug}.md ↔ Updates: Ideas/{slug}.md (status) ↔ Updates: Goals/2026-Q2.md (link)
Stage 7 Build cd ~/projects/{slug}/

Open Claude Code in the project. CLAUDE.md gives full context. Agents are available. Skills can be added. Meetings auto-link via Dataview.

project code skills added as needed agents refined
Obsidian I/O ← Reads: project note (via CLAUDE.md link) ← Dataview: auto-surfaces meetings, ideas, people
